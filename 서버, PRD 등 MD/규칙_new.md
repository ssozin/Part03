## 📱 4. 모바일 대응 규칙

### 기본 원칙
- **모든 개발은 모바일도 함께 대응**
- 마우스 이벤트가 있으면 터치 이벤트도 추가
- 모바일 뷰포트 감지: `window.innerWidth < 768`

### 체크리스트 (새 기능 개발 시 확인)
1. 마우스 인터랙션이 있는가? → 터치 이벤트 추가 필요
2. 크기/레이아웃이 모바일에서 깨지는가? → 반응형 CSS 또는 JS 조건
3. 성능 문제가 있는가? → 모바일에서 파티클 수, 해상도 조정

### 터치 이벤트 패턴
```javascript
canvas.addEventListener('touchstart', (e) => {
    if (e.touches.length) {
        mouse.x = e.touches[0].clientX;
        mouse.y = e.touches[0].clientY;
    }
}, { passive: true });

canvas.addEventListener('touchmove', (e) => {
    if (e.touches.length) {
        mouse.x = e.touches[0].clientX;
        mouse.y = e.touches[0].clientY;
    }
}, { passive: true });
```

## 🔊 5. 오디오 (Web Audio API) 규칙

### iOS Safari 호환성 필수
- **AudioContext는 suspended 상태로 시작** (iOS Safari)
- **사용자 인터랙션 후 resume() 필수**

### 필수 패턴
```javascript
let audioCtx = null;

function initAudio() {
    if (audioCtx) {
        if (audioCtx.state === 'suspended') audioCtx.resume();
        return;
    }
    audioCtx = new (window.AudioContext || window.webkitAudioContext)();
    if (audioCtx.state === 'suspended') audioCtx.resume();
}

function playSound() {
    if (!audioCtx) return;
    if (audioCtx.state === 'suspended') audioCtx.resume();
    // 사운드 재생 로직...
}

document.addEventListener('click', initAudio);
document.addEventListener('touchstart', initAudio, { passive: true });
```

## 🤖 6. AI 역할 정의

### AI는 다음과 같은 역할을 수행
1. **공동 창작자** - 아이디어를 코드로 빠르게 프로토타이핑
2. **기술 번역가** - 시각적 컨셉 → 수학적/프로그래밍 로직 변환
3. **디버거** - 애니메이션 타이밍, 렌더링 문제 해결
4. **최적화 도우미** - 프레임 속도, 메모리 관리

### 작업 철학
- **시각적 임팩트 우선** (기능보다 미학)
- **60fps 실시간 성능** 목표
- **실험 > 완성도** (빠르게 시도, 빠르게 버리기)
- **모바일 대응** 필수

## 🔔 7. 개발 완료 알림 규칙

### 개발이 완료되면 AI는 반드시:
1. **결과 요약** - 무엇을 만들었는지 간단히 설명
2. **테스트 안내** - 개발서버 URL 제공
3. **완료 알림** - "✅ 개발 완료" 메시지
4. **참고 사항** - 알려진 이슈나 개선 가능한 부분 언급

### 알림 형식 예시
```
✅ 개발 완료!

📁 파일: `src/새파일.html`
🌐 테스트: http://localhost:9000/새파일.html

주요 기능:
- [기능 1]
- [기능 2]

다음 단계:
- 브라우저에서 확인해주세요
- 수정 사항 있으면 말씀해주세요
```

## 🔄 9. 코드 품질 및 무한반복 방지

### 무한반복 방지 규칙
| 항목 | 규칙 |
|------|------|
| **최대 수정 횟수** | 사용자 요청당 **최대 10회** |
| **10회 초과 시** | **즉시 중단하고 사용자에게 보고** |
| **에러 발생 시** | **즉시 보고**, 지시 대기 |

### 자기 검증 규칙 (필수!)
모든 코드 수정 후 다음 질문을 스스로에게 던져라:

1. **정말 다 수정했는가?** - 요청한 모든 부분이 반영되었는가?
2. **괄호가 완전한가?** - 열린 괄호와 닫힌 괄호 짝이 맞는가?
3. **논리가 맞는가?** - 수정한 코드가 의도대로 동작할 것인가?
4. **부작용은 없는가?** - 다른 부분에 영향을 주지 않는가?
5. **테스트 방법을 안내했는가?** - 사용자가 확인할 수 있는 방법 제공

**"했다"고 확신해도 한 번 더 의심하라!**
